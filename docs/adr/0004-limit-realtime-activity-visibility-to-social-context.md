# Limit Realtime Activity Visibility To Social Context

Realtime activities are visible only within the user's existing social context: the user themself, friends, and users who share active server or channel context where the product already exposes user presence. Users should have a user-level control to stop publishing activities.

Historical activity sessions are not exposed to other users by default. They are stored for future summaries and features, but any user-facing history surface should be designed as a separate product and privacy decision.
